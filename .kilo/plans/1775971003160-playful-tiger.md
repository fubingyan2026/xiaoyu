# LED GPIO/PWM 切换问题 - 再次故障排查

## 问题描述

PWM 显示正常切换（函数返回成功），但 LED 完全不亮。

## 问题分析

用户配置：min_duty = 100-500，max_duty 应该 > min_duty
现象：切换到呼吸模式后 LED 完全不亮

可能原因：
1. **MOE 位未使能**: 高级定时器 TIM17 需要 MOE 位使能才能输出 PWM
2. **Alternate 配置错误**: 虽然函数返回成功，但 AF 号可能配置错误
3. **GPIO 被复用**: `hal_gpio_init` 可能重新配置了引脚模式，覆盖了 PWM 设置

## 解决方案

### 方案：强制确保 MOE 位使能 + 简化 Entry 逻辑

在 `led_fsm_on_entry` 中，进入呼吸灯时**强制重新启动定时器并使能 MOE**：

```c
case LED_STATE_BREATHING:
  handle->breath_value = handle->config.led_refresh_min_duty;
  handle->breath_cycle = 0;
  handle->last_breath_time = now;

  if (handle->pwm_init_flag) {
    // 1. 先停止 PWM（确保干净的状态）
    hal_tim_pwm_stop(&tim_pwm_ctx, handle->config.pwm_cfg.timer_instance,
                     handle->config.pwm_cfg.channel);

    // 2. 设置初始占空比
    hal_tim_pwm_set_duty_cycle(&tim_pwm_ctx, handle->config.pwm_cfg.timer_instance,
                               handle->config.pwm_cfg.channel, handle->breath_value);

    // 3. 切换为 PWM 复用模式
    hal_tim_pwm_gpio_alternate(&tim_pwm_ctx, &handle->config.pwm_cfg.gpio);

    // 4. 重新启动 PWM（确保 MOE 位使能）
    hal_tim_pwm_start(&tim_pwm_ctx, handle->config.pwm_cfg.timer_instance,
                      handle->config.pwm_cfg.channel);
  }
  break;
```

### 修改文件

- `led/led.c` - `led_fsm_on_entry` 的 LED_STATE_BREATHING 处理

### 风险说明

之前的方案是"定时器劫持法"（只启动一次，永不停止），这次改为每次进入呼吸灯都重新启动。

这样做的影响：
- 每次切换都需要 Start/Stop，可能有轻微性能开销
- 但能确保定时器状态干净，MOE 位正确使能

**注意**: 如果这样还是不行，问题可能在 `hal_tim_pwm_gpio_alternate` 的底层实现，需要检查 AF 号配置是否正确。
