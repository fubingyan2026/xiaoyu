# FOC驱动 串口通信协议规范

**文档版本**: V2.0  
**参考规范**: 西门子 S7comm 工业通信标准  
**日期**: 2026-03-03

---

## 一、协议概述

本协议参考西门子S7comm协议规范设计,适用于FOC电机驱动与上位机之间的串口通信。

### 1.1 主要特性

| 特性 | 说明 |
|------|------|
| 协议ID | 0x32 (参考S7协议) |
| 校验方式 | CRC16 Modbus |
| 字节序 | Little-Endian (小端序) |
| 帧格式 | 固定头 + 变长数据 + 校验尾 |

---

## 二、帧格式

### 2.1 基本帧结构

```
┌──────────┬──────────┬──────────┬──────────┬──────────┬──────────┐
│  帧头    │ 协议ID   │ 消息类型 │  参数    │  数据    │  校验    │
│  2字节   │  1字节   │  1字节   │  1字节   │  N字节   │  2字节   │
└──────────┴──────────┴──────────┴──────────┴──────────┴──────────┘
```

### 2.2 帧字段详细说明

| 字段 | 长度 | 说明 | 必填 |
|------|------|------|------|
| 帧头 | 2B | 固定 `0x68 0x68` | 是 |
| 协议ID | 1B | 固定 `0x32` | 是 |
| 消息类型 | 1B | 请求/响应/错误 | 是 |
| 参数区 | 1B | 功能码/子功能码 | 是 |
| 数据区 | 0-240B | 实际数据 | 否 |
| CRC16 | 2B | Modbus CRC16 | 是 |
| 帧尾 | 1B | 固定 `0x16` | 是 |

**最小帧长**: 8字节  
**最大帧长**: 255字节

## 三、消息类型

| 消息类型 | 代码 | 说明 | 方向 |
|----------|------|------|------|
| JOB | 0x01 | 请求/命令 | PC→MCU |
| ACK | 0x02 | 简单确认 (无数据) | MCU→PC |
| RESPONSE | 0x03 | 带数据的响应 | MCU→PC |
| ERROR | 0x07 | 错误响应 | MCU→PC |

**流数据类型**:

## 七、CRC16校验

### 7.1 CRC16 Modbus计算

```c
static const uint16_t crc16_table[256] = {
    0x0000, 0xC0C1, 0xC181, 0x0140, 0xC301, 0x03C0, 0x0280, 0xC241,
    0xC601, 0x06C0, 0x0780, 0xC741, 0x0500, 0xC5C1, 0xC481, 0x0440,
    // ... (完整256字节表)
};

uint16_t crc16_modbus(const uint8_t *data, uint16_t len)
{
    uint16_t crc = 0xFFFF;
    for (uint16_t i = 0; i < len; i++) {
        crc = (crc >> 8) ^ crc16_table[(crc ^ data[i]) & 0xFF];
    }
    return crc;
}
```

### 7.2 校验示例

**请求帧** `PING`:
```
原始数据: 68 32 01 01 01
CRC16:    5A 6B
完整帧:  68 32 01 01 01 5A 6B 16
```

## 十二、注意事项

1. **字节序**: 所有多字节数据采用小端序 (Little-Endian)
2. **数据范围**: 发送前请检查数据是否在有效范围内
3. **流数据**: 开始流数据后,MCU会定期自动发送数据帧
4. **超时处理**: 建议PC端设置500ms超时时间
5. **重试机制**: 建议最多重试3次
6. **CRC校验**: 所有帧都必须进行CRC16校验
