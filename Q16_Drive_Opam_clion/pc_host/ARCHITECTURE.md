# FOC上位机工业级架构设计方案

## 1. 概述

### 1.1 文档目的

本文档定义了FOC电机驱动上位机从基础版本升级到工业级标准的完整架构设计方案。

### 1.2 目标用户

- 工业自动化产线工程师
- 电机调试与维护技术人员
- 设备集成商

### 1.3 设计原则

| 原则 | 说明 | 优先级 |
|------|------|--------|
| **可靠性** | 7×24小时稳定运行，通信故障自恢复 | P0 |
| **可追溯** | 所有操作和数据可审计、可回溯 | P0 |
| **安全性** | 权限分级，危险操作受控 | P0 |
| **可维护** | 模块化设计，故障快速定位 | P1 |
| **可扩展** | 支持多设备、多协议 | P1 |

## 3. 核心模块详细设计

### 3.1 通信管理模块 (Communication Manager)

#### 3.1.1 功能要求

| 功能 | 说明 | 指标 |
|------|------|------|
| 多协议支持 | A5协议、Modbus、CAN | 可配置切换 |
| 自动重连 | 检测断线后自动恢复 | <3秒恢复 |
| 连接池 | 同时管理多台设备 | 最多16台 |
| 通信质量 | 统计丢包率、延迟 | 实时监控 |
| 发送队列 | 命令排队和优先级 | 256条缓冲 |

#### 3.1.2 状态机设计

| 状态 | 触发条件 | 行为 |
|------|----------|------|
| DISCONNECTED | 初始化/断开 | 等待连接命令 |
| CONNECTING | 收到连接请求 | 尝试建立连接，超时5秒 |
| CONNECTED | 连接成功 | 启动心跳、接收数据 |
| RECONNECTING | 检测到断线 | 指数退避重试(1s→2s→4s→8s→30s) |
| ERROR | 连续失败3次 | 通知用户，停止重连 |

#### 3.1.3 类结构

```python
class CommunicationManager:
    - ConnectionPool connections    # 连接池
    - MessageQueue tx_queue         # 发送队列
    - ProtocolFactory protocols     # 协议工厂
    + connect(device_config)        # 建立连接
    + disconnect(device_id)         # 断开连接
    + send_command(device_id, cmd)  # 发送命令
    + register_callback(event, handler)

class ReliableConnection:
    - SerialPort port
    - ProtocolHandler protocol
    - StateMachine state
    - HeartbeatMonitor heartbeat
    - Statistics stats
    + reconnect()                   # 自动重连
    + get_quality_report()          # 通信质量报告
```

### 3.2 数据记录模块 (Data Logger)

#### 3.2.1 数据库设计

**表结构**:

| 表名 | 用途 | 字段 |
|------|------|------|
| `runtime_data` | 实时运行数据 | id, timestamp, device_id, current_d, current_q, velocity, position, voltage, temp |
| `alarms` | 告警记录 | id, timestamp, device_id, level, code, message, ack_time, ack_user |
| `operations` | 操作日志 | id, timestamp, user_id, device_id, action, params, result |
| `calibration` | 校准记录 | id, timestamp, device_id, params, result, data_file |
| `devices` | 设备信息 | id, name, serial_no, model, config_file, created_at |
| `configs` | 配置版本 | id, device_id, version, params, created_by, created_at |

#### 3.2.2 数据存储策略

| 数据类型 | 存储方式 | 保留周期 | 压缩策略 |
|----------|----------|----------|----------|
| 实时数据 | SQLite + 内存缓冲 | 7天 | 1小时后降采样(1:10) |
| 告警数据 | SQLite | 1年 | 不压缩 |
| 操作日志 | SQLite + 文件日志 | 3年 | 按月归档 |
| 校准数据 | 文件系统(JSON) | 永久 | 不压缩 |
| 波形数据 | 二进制文件 | 30天 | 自动清理 |

#### 3.2.3 数据导出格式

支持格式:

| 格式 | 扩展名 | 适用场景 |
|------|--------|----------|
| CSV | .csv | Excel分析 |
| Excel | .xlsx | 报表生成 |
| MAT | .mat | MATLAB分析 |
| JSON | .json | 程序处理 |
| 二进制 | .bin | 高效存储 |

### 3.3 告警管理模块 (Alarm Manager)

#### 3.3.1 告警级别

| 级别 | 颜色 | 响应时间 | 通知方式 | 示例 |
|------|------|----------|----------|------|
| INFO | 蓝色 | 无 | 日志记录 | 参数修改 |
| WARNING | 黄色 | <1分钟 | 界面提示 | 电流偏高 |
| ALARM | 橙色 | <10秒 | 弹窗+声音 | 过温警告 |
| EMERGENCY | 红色 | 立即 | 全屏+声音+邮件 | 过流保护 |

#### 3.3.2 告警规则配置

```yaml
alarms:
  overcurrent:
    name: "过流保护"
    condition: "abs(current_q) > 5.0"
    level: "EMERGENCY"
    delay: 0
    action: "stop_motor"
    
  overtemp:
    name: "过温警告"
    condition: "temperature > 80"
    level: "ALARM"
    delay: 5000  # ms
    action: "reduce_current"
    
  velocity_deviation:
    name: "速度偏差"
    condition: "abs(target_velocity - actual_velocity) > 0.1"
    level: "WARNING"
    delay: 1000
    action: "log_only"
```

### 3.4 配置管理模块 (Config Manager)

#### 3.4.1 配置层级

| 层级 | 优先级 | 说明 | 存储位置 |
|------|--------|------|----------|
| 系统默认 | 最低 | 出厂设置 | 代码内置 |
| 设备模板 | 中 | 同型号设备配置 | config/templates/ |
| 设备专属 | 高 | 单台设备参数 | config/devices/{id}.yaml |
| 运行时 | 最高 | 临时修改 | 内存 |

  control:

      kp: 0.1

      kp: 10.0

### 3.6 固件升级模块 (Firmware Updater)

#### 3.6.1 升级流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  选择   │───→│  校验   │───→│  擦除   │───→│  写入   │

│  完成   │←───│  验证   │←───│  重启   │←──────┘

```

#### 3.6.2 安全机制

| 机制 | 说明 |
|------|------|
| 版本校验 | 新固件版本必须高于当前版本 |
| CRC校验 | 传输前后双重校验 |
| 双区备份 | 保留旧固件，失败可回滚 |
| 断电保护 | 写Flash期间断电不会变砖 |
| 签名验证 | 固件需要数字签名 |
