# FOC电机驱动上位机通信协议

## 概述

本文档描述PC上位机与STM32 FOC电机驱动板之间的串口通信协议。

- **协议版本**: v1.0
- **物理层**: UART
- **波特率**: 115200 (默认)
- **数据位**: 8位
- **停止位**: 1位
- **校验**: 无

---

## 帧格式

### 基本帧结构

| 字段 | 长度 | 说明 |
|------|------|------|
| 帧头 | 2 bytes | `0xA5 0x5A` |
| 命令码 | 1 byte | 命令类型 |
| 数据长度 | 1 byte | Payload长度 (0-255) |
| 数据 | n bytes | 实际数据内容 |
| 校验和 | 1 byte | XOR校验 |
| 帧尾 | 2 bytes | `0x0D 0x0A` (\\r\\n) |

**总长度**: 7 + n bytes (n为数据长度)

### 校验和计算

```c
uint8_t checksum = cmd ^ len;
for (int i = 0; i < len; i++) {
    checksum ^= data[i];
}
```

## 命令列表

### 1. 系统命令

| 命令 | 代码 | 方向 | 数据长度 | 说明 |
|------|------|------|----------|------|
| CMD_PING | 0x01 | PC→MCU | 0 | 心跳/连接测试 |
| CMD_GET_VERSION | 0x02 | PC→MCU | 0 | 获取固件版本 |
| CMD_RESET | 0x03 | PC→MCU | 0 | 系统复位 |
| RESP_OK | 0x80 | MCU→PC | 0 | 通用成功响应 |
| RESP_ERROR | 0x81 | MCU→PC | 1 | 错误响应，数据为错误码 |
| RESP_DATA | 0x82 | MCU→PC | 变长 | 数据响应 |

### 2. 状态查询命令

| 命令 | 代码 | 方向 | 数据长度 | 说明 |
|------|------|------|----------|------|
| CMD_GET_STATUS | 0x10 | PC→MCU | 0 | 获取当前运行状态 |
| CMD_GET_STATE | 0x11 | PC→MCU | 0 | 获取FOC状态机状态 |

**响应数据格式 (CMD_GET_STATUS)**:

| 字节 | 字段 | 类型 | 说明 |
|------|------|------|------|
| 0 | state | uint8 | FOC状态机状态 |
| 1 | error_code | uint8 | 错误代码 |
| 2-3 | Id | int16 | D轴电流 (mA) |
| 4-5 | Iq | int16 | Q轴电流 (mA) |
| 6-7 | velocity | int16 | 速度 (rad/s × 1000) |
| 8-11 | position | int32 | 位置 (rad × 10000) |
| 12-13 | voltage | uint16 | 母线电压 (V × 100) |
| 14 | temperature | int8 | 温度 (°C + 40) |

### 3. 控制命令

| 命令 | 代码 | 方向 | 数据长度 | 说明 |
|------|------|------|----------|------|
| CMD_SET_CURRENT | 0x20 | PC→MCU | 4 | 设置电流 (Id, Iq) |
| CMD_SET_VELOCITY | 0x21 | PC→MCU | 2 | 设置速度 |
| CMD_SET_POSITION | 0x22 | PC→MCU | 4 | 设置位置 |
| CMD_STOP | 0x23 | PC→MCU | 0 | 紧急停止 |

**CMD_SET_CURRENT 数据格式**:

| 字节 | 字段 | 类型 | 范围 | 单位 |
|------|------|------|------|------|
| 0-1 | Id | int16 | -10000 ~ 10000 | 0.001A |
| 2-3 | Iq | int16 | -10000 ~ 10000 | 0.001A |

**CMD_SET_VELOCITY 数据格式**:

| 字节 | 字段 | 类型 | 范围 | 单位 |
|------|------|------|------|------|
| 0-1 | velocity | int16 | -32768 ~ 32767 | 0.001 rad/s |

**CMD_SET_POSITION 数据格式**:

| 字节 | 字段 | 类型 | 范围 | 单位 |
|------|------|------|------|------|
| 0-3 | position | int32 | -2147483648 ~ 2147483647 | 0.0001 rad |

### 4. 编码器校准命令

| 命令 | 代码 | 方向 | 数据长度 | 说明 |
|------|------|------|----------|------|
| CMD_START_CALIB | 0x30 | PC→MCU | 0 | 开始编码器校准 |
| CMD_GET_CALIB_STATUS | 0x31 | PC→MCU | 0 | 获取校准状态 |
| CMD_GET_CALIB_DATA | 0x32 | PC→MCU | 0 | 获取校准数据 |
| CMD_SET_CALIB_DATA | 0x33 | PC→MCU | 变长 | 写入校准数据 |
| CMD_CLEAR_CALIB | 0x34 | PC→MCU | 0 | 清除校准数据 |

**校准状态响应 (CMD_GET_CALIB_STATUS)**:

| 字节 | 字段 | 类型 | 说明 |
|------|------|------|------|
| 0 | status | uint8 | 校准状态 |
| 1 | progress | uint8 | 进度百分比 (0-100) |
| 2 | direction | int8 | 方向 (-1=反向, 1=正向) |

**校准状态码**:

| 状态码 | 名称 | 说明 |
|--------|------|------|
| 0 | IDLE | 空闲/就绪 |
| 1 | RUNNING | 正在校准 |
| 2 | FORWARD | 正向旋转中 |
| 3 | REVERSE | 反向旋转中 |
| 4 | COMPLETE | 校准完成 |
| 5 | FAILED | 校准失败 |

**校准数据格式 (CMD_GET_CALIB_DATA 响应)**:

| 字节 | 字段 | 类型 | 说明 |
|------|------|------|------|
| 0-1 | angle_map[0] | uint16 | 第1个扇区角度值 |
| 2-3 | angle_map[1] | uint16 | 第2个扇区角度值 |
| ... | ... | ... | ... |
| n-(n+1) | angle_map[N-1] | uint16 | 第N个扇区角度值 |
| n+2 | direction | int8 | 编码器方向 |

> 注: N = 电机极对数 × 4，例如11极对电机，N=44

## 通信示例

### 示例1: 心跳检测

**PC发送**:
```
A5 5A 01 00 01 0D 0A
```
| 字节 | 值 | 说明 |
|------|-----|------|
| 帧头 | A5 5A | 固定帧头 |
| 命令 | 01 | CMD_PING |
| 长度 | 00 | 无数据 |
| 校验 | 01 | 0x01 ^ 0x00 = 0x01 |
| 帧尾 | 0D 0A | 固定帧尾 |

**MCU响应**:
```
A5 5A 80 00 80 0D 0A
```
| 字节 | 值 | 说明 |
|------|-----|------|
| 帧头 | A5 5A | 固定帧头 |
| 命令 | 80 | RESP_OK |
| 长度 | 00 | 无数据 |
| 校验 | 80 | 0x80 ^ 0x00 = 0x80 |
| 帧尾 | 0D 0A | 固定帧尾 |

### 示例2: 设置电流 (Id=0.5A, Iq=1.0A)

**数据转换**:
- Id = 0.5A → 500 (0x01F4)
- Iq = 1.0A → 1000 (0x03E8)

**PC发送**:
```
A5 5A 20 04 F4 01 E8 03 1F 0D 0A
```
| 字节 | 值 | 说明 |
|------|-----|------|
| 帧头 | A5 5A | 固定帧头 |
| 命令 | 20 | CMD_SET_CURRENT |
| 长度 | 04 | 4字节数据 |
| 数据 | F4 01 | Id = 500 (小端) |
| 数据 | E8 03 | Iq = 1000 (小端) |
| 校验 | 1F | 计算: 0x20^0x04^0xF4^0x01^0xE8^0x03 = 0x1F |
| 帧尾 | 0D 0A | 固定帧尾 |

### 示例3: 开始数据流

**PC发送**:
```
A5 5A 50 01 00 51 0D 0A
```
| 字节 | 值 | 说明 |
|------|-----|------|
| 帧头 | A5 5A | 固定帧头 |
| 命令 | 50 | CMD_START_STREAM |
| 长度 | 01 | 1字节数据 |
| 数据 | 00 | stream_type = STREAM_ALL |
| 校验 | 51 | 0x50^0x01^0x00 = 0x51 |
| 帧尾 | 0D 0A | 固定帧尾 |

**MCU持续发送流数据**:
```
A5 5A 83 10 [16字节数据] [校验] 0D 0A
```

## 注意事项

1. **字节序**: 所有多字节数据采用小端序 (Little-Endian)
2. **数据范围**: 发送前请检查数据是否在有效范围内
3. **流数据**: 开始流数据后，MCU会定期自动发送数据帧
4. **超时处理**: 建议PC端设置500ms超时时间
5. **重试机制**: 建议最多重试3次
